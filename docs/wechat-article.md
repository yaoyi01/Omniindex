## 🎯 它能做什么？一句话

**在几十万文件中，用自然语言搜索「财务报表」，秒级返回所有相关文件——哪怕文件名里根本没出现这四个字。**

---

## 🧠 怎么做到的？

OmniIndex 由三个部分组成：

### 1. 终端采集 Agent（不到 2MB）

在每台 Windows 机器上运行一个轻量级采集程序。它**不扫磁盘、不装驱动、不用管理员权限**——直接读取 Windows 自带的 Search Index，获取文件名、路径、修改时间等元数据，加密上传到中心服务器。

```
Agent 架构：
  Windows Search Index (SystemIndex)
       ↓ OLE DB 查询
  采集程序 (.NET 4.8)
       ↓ HTTP 批量上传
  中心服务器
```

### 2. AI 向量化引擎

文件元数据到达服务器后，通过阿里云 DashScope 的 `text-embedding-v3` 模型转换为 **1024 维稠密向量**。这不是简单的"建立索引"——它把文字映射到高维语义空间，让"财务报表"和"利润表"的距离比"财务报表"和"游戏攻略"近一千倍。

### 3. LLM 驱动的语义搜索

这是最核心的一环：

```
你输入："财务报表"
  ↓
DeepSeek LLM 自动理解意图，扩展关键词：
  → 财务报表、利润表、P&L、balance sheet、income statement...
  ↓
同时生成优化的语义检索文本：
  → "财务报表相关文档，包括利润表、资产负债表、现金流量表等"
  ↓
两端并发：关键字匹配 + 向量余弦相似度检索
  ↓
合并去重，按相关度排序返回
```

**搜关键词找不到的，用语义找。**

## 📊 管理后台长什么样

一个暗色主题的 Web 仪表盘，一眼看完：

- **文件数量**：实时计数，带动态数字滚动动画
- **终端状态**：每台机器的在线/离线、文件数、向量化率
- **文件类型分布**：饼图展示代码/文档/图片/数据库的占比
- **向量化覆盖率**：柱状图对比已处理 vs 待处理
- **终端文件浏览器**：点击任意终端，直接预览它采集了哪些文件
- **系统设置**：远程调整采集间隔、文件类型过滤

**不需要 SSH 到服务器，浏览器里全部搞定。**

## ⚡ 5 分钟跑起来

### 1. 启动后端（一行命令）

```bash
git clone https://github.com/yaoyi01/Omniindex
cd Omniindex
cp server/.env.example server/.env
# 编辑 .env 填入 API Key
./start.sh
```

### 2. 部署采集 Agent（一行命令）

```powershell
cd agent
dotnet publish -c Release -o dist
# 把 dist/desktop-search-agent.exe 拷到目标机，同目录创建 config.json
```

### 3. 打开浏览器

`http://localhost:5173`，开始搜索。

## 🔧 技术栈一览

| 层 | 技术选型 | 为什么 |
|---|---|---|
| Agent | C# .NET 4.8 | Windows Search OLE DB 原生支持，单文件打包 |
| 后端 | Python FastAPI | 异步高性能，Swagger 文档自动生成 |
| 数据库 | SQLite（可切 PG） | 开箱即用，1.5 亿条也没问题 |
| 向量化 | DashScope text-embedding-v3 | 1024 维，中文效果远超开源模型 |
| 搜索增强 | DeepSeek Chat | 中文语义理解天花板，查询扩展精准 |
| 前端 | React + Tailwind + Recharts | 暗色主题，响应式，出活快 |

## ⚠️ 诚实地说局限性

作为 MVP（最小可用产品），它不完美：

1. **只支持 Windows 终端** — Agent 依赖 Windows Search Index，Linux/Mac 用不了
2. **文件内容索引受限** — 图片、加密文档的文本摘要可能为空，搜索结果依赖文件名
3. **没有用户认证** — 当前版本无登录/权限控制，适合内网单用户或全员可信环境
4. **向量模型依赖云 API** — DashScope 挂了就回退为纯关键字搜索，失去语义能力
5. **大事务删除慢** — SQLite 模式下删除一个终端的所有文件需要分批执行
6. **没有增量采集** — 每次采集全量扫描，不做 LastModifiedTime 比对（下个版本会加）
